## 2. Matching CV × Offres (100% local, sans API IA)

Convertir les offres et un CV en Markdown, puis calculer un score de matching par similarité textuelle simple (TF-IDF ou Jaccard sur les mots-clés).

### Principe

```
CV (PDF ou texte)  ──→  cv.md  (texte propre)
Offre Supabase     ──→  offer-{id}.md  (titre + company + tags + description)
                              ↓
                   score_matching(cv.md, offer.md)
                   = % de mots-clés communs pondérés par fréquence
```

Pas de call API IA — tout tourne en Python pur dans le scraper ou dans une nouvelle commande CLI.

### Fichiers à créer

| Fichier | Rôle |
|---------|------|
| `tools/cv_to_md.py` | Extrait le texte d'un PDF CV et le convertit en `.md` propre |
| `tools/offer_to_md.py` | Récupère les offres Supabase et génère des fichiers `.md` |
| `tools/matcher.py` | Calcule le score TF-IDF entre `cv.md` et chaque `offer.md`, produit un classement |
| `tools/run_matching.py` | Script CLI : `python tools/run_matching.py --cv mon_cv.pdf` |
| `tools/requirements.txt` | `pdfminer.six`, `scikit-learn` (TF-IDF), `supabase` |

### Algorithme matcher (sans IA)

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

def match(cv_text: str, offers: list[dict]) -> list[dict]:
    docs = [cv_text] + [o['text'] for o in offers]
    tfidf = TfidfVectorizer(stop_words=None)  # garder les termes techniques FR
    matrix = tfidf.fit_transform(docs)
    scores = cosine_similarity(matrix[0:1], matrix[1:])[0]
    return sorted(
        [{'offer': o, 'score': round(float(s) * 100, 1)} for o, s in zip(offers, scores)],
        key=lambda x: x['score'],
        reverse=True
    )
```
